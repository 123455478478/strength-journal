# 个人力量训练记录 App 软件设计

## 1. 设计范围

产品只负责记录实际训练，不提供周计划、课程安排和训练建议。

主导航为：**训练 / 动作 / 历史 / 我的**。

## 2. 核心页面

### 训练首页

- 没有进行中的训练时，显示“开始健身”；点击后立即保存训练开始时间并启动总计时。
- 有未完成训练时，显示“继续训练”、已用时间、动作数和完成组数。
- 已用时间每秒同步更新，“继续健身”旁提供红色“结束健身”快捷按钮。
- “建议动作”在第一组开始前避开上一次训练部位，并参考前两次覆盖情况；第一组开始后只推荐该肌群相关动作。候选动作再按最近八次使用偏好排序。
- 最近记录展示日期、主要肌群、动作及各动作完成组数，不展示下一次训练安排。

### 自由训练

- 初始为空，通过“添加动作”从动作库选择。
- 未开始健身时只能浏览动作，不能向训练添加动作。
- 动作可任意排序，不限制数量。
- 每个动作显示名称、肌群、完成组数和最高重量。
- 训练中可以随时添加、删除或跳过动作。

### 动作记录

- 第一组开始前显示上一次该动作的表现。
- 每组点击“开始本组”和“结束本组”；结束时先固定时间，再进入重量和次数填写页。
- 填写页提供重量 ±1 和次数 ±1 快捷调整；新组沿用上一组，第一组读取最近同动作第一组。
- 未开始的额外组支持左滑删除。
- 完成本组后实时显示不断增加的组间间歇。
- 开始下一组后固定上一段间歇。
- 动作总用时取第一组开始到最后一组结束。
- 结束训练后的总结页提供自动保存的文字心得输入框。

### 动作库

- 一级筛选为胸部、背部、肩部、手臂、腿和核心，并在每个部位下按动作模式或细分肌群分组。
- 按名称、细分部位、肌群和器械搜索。
- 支持收藏、最近使用和自定义动作。
- 动作详情包含名称、肌群、器械、负重倾向、常用次数和安全要点。

### 历史

- 月历中标记存在训练记录的日期。
- 月历下按时间倒序排列记录。
- 按日期展示训练次数、总用时、动作数和组数。
- 单次记录展示每个动作的重量、次数、每组用时和组间间歇。
- 点击日历日期进入当日详情，可新增当日训练，或编辑、删除已有训练及其动作和每组数据。
- 记录卡片左滑露出删除按钮，确认后删除。
- 动作趋势展示历次最高重量、总次数和完成组数。

### 我的

- 记录身高、体重、年龄和性别。
- 记录卧推/深蹲 1RM，以及引体向上/俯卧撑连续次数极限。
- kg/lb 单位切换。
- JSON 完整备份与恢复。
- CSV 训练记录导出。
- 清除数据必须二次确认。

## 3. 训练状态

```text
未训练
  → 开始健身 / 启动训练总计时
训练为空
  → 添加动作
动作待开始
  → 开始本组
本组进行中
  → 结束本组
本组数据待填写
  → 填写重量、次数并保存
组间歇进行中
  → 开始下一组
动作完成或继续加组
  → 添加其他动作 / 结束健身
训练总结
  → 保存历史
```

## 4. 视觉方向

- 浅灰白背景、深绿色主操作色、琥珀色计时状态。
- 训练进行时隐藏底部导航，避免误触。
- 重量、次数和时间使用等宽数字。
- 不使用成长等级、连续打卡压力或课程推广。

## 5. 数据可靠性

- 状态改变立即写入本地数据库。
- 计时基于绝对时间戳，不依赖页面保持前台。
- 训练历史保存动作资料快照。
- 所有自动计算结果都可以由原始开始/结束时间重新生成。

## 6. 动作资料参考

- 负荷与次数倾向参考 ACSM 2026 年阻力训练指南的一般原则，并按复合动作、孤立动作和动作控制需求做界面化归纳。
- 动作执行与安全提示参考 ACE Exercise Library 的公开动作说明，再改写为简短中文要点。
- 这些内容用于一般信息展示，不替代教练评估、康复指导或医疗建议。

## 7. 部署形态

- 静态 PWA，不要求构建步骤或服务端数据库。
- Manifest 支持添加到手机主屏幕。
- Service Worker 缓存应用外壳，首次联网打开后可离线再次进入。
- 用户数据保存在当前浏览器本机，部署不会自动获得跨设备同步能力。
